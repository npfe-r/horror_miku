# 恐怖游戏设计文档 - 夜校逃亡

## 1. 项目概述

### 1.1 基本信息
- **项目名称**: 夜校逃亡 (Horror Miku)
- **引擎版本**: Godot 4.6
- **游戏类型**: 第一人称恐怖生存
- **目标平台**: PC (Windows)

### 1.2 游戏简介
玩家扮演一名被困在夜间学校的学生，学校中有一个可怕的怪物四处游荡。玩家需要探索学校、收集钥匙和道具、解开机关谜题，最终找到出口逃离学校。在探索过程中，玩家需要躲避怪物的追击，利用藏身处和道具来保护自己。

### 1.3 核心玩法循环
1. **探索** - 在学校中移动，寻找线索和道具
2. **躲避** - 听到或看到怪物时，寻找藏身处躲避
3. **解谜** - 使用道具和机关打开新的区域
4. **逃脱** - 收集所有必要物品，找到出口逃离

## 3. 怪物AI设计

### 3.1 怪物行为状态机
```
┌─────────────┐
│   巡逻      │ ←──────────────────────┐
└──────┬──────┘                        │
       │ 听到声音/看到玩家              │
       ▼                               │
┌─────────────┐                        │
│   警觉      │ ──失去目标─────────────┘
└──────┬──────┘
       │ 确认玩家位置
       ▼
┌─────────────┐
│   追击      │
└──────┬──────┘
       │ 玩家进入藏身处/距离过远
       ▼
┌─────────────┐
│   搜索      │ ──搜索失败─────────────┐
└─────────────┘                        │
                                       │
              返回巡逻 ←────────────────┘
```

### 3.2 状态详细说明

#### 3.2.1 巡逻状态 (Patrol)
- 沿预设路径点移动
- 移动速度: 2.0 m/s
- 在每个路径点停留 2-5 秒
- 持续监听周围噪音
- 视野范围: 前方 120° 扇形区域，距离 15m

#### 3.2.2 警觉状态 (Alert)
- 停止移动，转向噪音来源方向
- 持续 3-5 秒观察
- 如果再次听到声音或看到玩家，进入追击状态
- 如果没有发现异常，返回巡逻状态

#### 3.2.3 追击状态 (Chase)
- 全速追向玩家最后已知位置
- 移动速度: 5.5 m/s (略快于玩家奔跑)
- 持续更新玩家位置
- 如果玩家进入藏身处，进入搜索状态
- 如果追击超过 30 秒仍未抓到玩家，进入搜索状态

#### 3.2.4 搜索状态 (Search)
- 在玩家最后已知位置周围搜索
- 搜索半径: 10m
- 搜索时间: 15-20 秒
- 检查附近的藏身处（有一定概率发现玩家）
- 搜索失败后返回巡逻状态

## 4. 交互系统设计

### 4.1 交互系统架构
```
InteractionManager (Autoload单例)
├── 当前可交互对象检测
├── 交互提示UI管理
└── 交互事件分发

IInteractable (接口)
├── interact() - 执行交互
├── get_interaction_text() - 获取交互提示文本
├── can_interact() - 检查是否可交互
└── get_highlight_material() - 获取高亮材质
```

### 4.2 交互检测
```gdscript
const INTERACT_RANGE: float = 2.5

func _physics_process(delta):
    var camera = get_viewport().get_camera_3d()
    var from = camera.project_ray_origin(get_viewport().get_mouse_position())
    var to = from + camera.project_ray_normal(get_viewport().get_mouse_position()) * INTERACT_RANGE
    
    var space_state = get_world_3d().direct_space_state
    var query = PhysicsRayQueryParameters3D.create(from, to)
    var result = space_state.intersect_ray(query)
    
    if result and result.collider.has_method("interact"):
        current_interactable = result.collider
        show_interaction_prompt(result.collider.get_interaction_text())
    else:
        current_interactable = null
        hide_interaction_prompt()
```

### 4.3 交互提示UI
- 屏幕中央显示交互提示文本
- 可交互物品高亮显示
- 不同类型物品显示不同图标

| 拼图 | 滑块/旋转拼图 | 完成拼图 |
| 电源 | 需要修复电路 | 找到并连接电线 |

### 5.3 藏身处系统

#### 5.3.1 藏身处类型
| 类型 | 隐藏效果 | 发现概率 | 特殊能力 |
|------|----------|----------|----------|
| 储物柜 | 高 | 低 | 可观察外部 |
| 桌底 | 中 | 中 | 快速进出 |
| 床底 | 高 | 低 | 无 |
| 窗帘后 | 低 | 高 | 可快速移动 |
| 通风管道 | 高 | 极低 | 可移动到其他房间 |

#### 5.3.2 藏身处属性
```gdscript
class_name HidingSpot
extends Node3D

@export var hiding_type: HidingType
@export var discovery_chance: float = 0.3  # 被怪物发现的基础概率
@export var can_observe: bool = false  # 是否可以观察外部
@export var connected_vent: NodePath  # 通风管道连接点

var occupant: Node3D

func enter(player: Node3D) -> bool
func exit() -> Node3D
func get_discovery_chance(monster_alert_level: float) -> float
```

#### 5.3.3 藏身机制
- 进入藏身处后，玩家无法移动（通风管道除外）
- 怪物搜索时有一定概率发现玩家
- 发现概率受以下因素影响：
  - 怪物警觉等级
  - 藏身处类型
  - 玩家进入藏身处时的噪音
  - 玩家是否在怪物视线内进入

### 5.4 其他可交互物品

#### 5.4.1 灯光开关
- 可以开关房间灯光
- 关灯可以降低被发现概率
- 怪物可能会重新开灯

#### 5.4.2 物品柜/抽屉
- 可以搜索获得道具
- 搜索会发出噪音
- 某些柜子需要撬开

#### 5.4.3 窗户
- 某些窗户可以打开
- 可以观察外部情况
- 部分窗户可以作为逃生路线

## 11. 游戏流程

### 11.1 游戏开始
1. 玩家在教学教室醒来
2. 简单的教学提示（移动、交互）
3. 发现自己被困在学校
4. 目标：找到出口钥匙，逃离学校

### 11.2 游戏进行
1. 探索学校，收集道具
2. 解开机关谜题，打开新区域
3. 躲避怪物追击
4. 收集所有必要的钥匙/卡片

### 11.3 游戏结局
- **成功逃脱**: 收集出口钥匙，打开大门
- **被怪物抓住**: 游戏结束，重新开始
